## 预期成果

三个独立报表从管理员导航可达，并通过权限、性能和全量回归验收。

## 背景

来源为 `../Token消耗统计页面实施Plan.md`。性能目标为统计 P95<1s、搜索 P95<500ms、响应建议<200KB；若环境不足，必须记录可复现的验证方案与限制。

## 范围内

- 添加三条前端路由、侧边栏分组、标题及中英文文案
- 执行前后端全量测试、构建和管理员权限回归
- 用大数据样本验证 P95、响应大小、分页上限及浏览器渲染
- 记录慢查询、索引结论和人工视觉验收

## 范围外

- 不新增导出、BI 或普通用户入口。

## 实现说明

- 保持管理员只读边界、项目全局时区和总 Token 既有口径。
- 所有列表必须有界；不要为了本 MVP 改动网关、计费或 Token 记账主链路。
